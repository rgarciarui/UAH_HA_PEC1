Introducción al dataset Fashion-Mnist¶

Jugando con la moda MNIST¶


Recientemente, los investigadores de Zalando, una empresa de comercio electrnico, presentaron a Fashion MNIST como un reemplazo directo del conjunto de datos original de MNIST. Al igual que MNIST, Fashion MNIST consiste en un conjunto de entrenamiento que consiste en 60.000 ejemplos pertenecientes a 10 clases diferentes y un conjunto de prueba de 10.000 ejemplos. Cada ejemplo de entrenamiento es una imagen en escala de grises, de tamao 28x28.

Entrar en la moda MNIST¶


Al publicar este conjunto de datos, los investigadores de Zalando hicieron las siguientes observaciones en MNIST:

  1. MNIST es demasiado fcil. Las redes convolucionales pueden alcanzar el 99.7% en MNIST. Los algoritmos clsicos de aprendizaje automtico tambin pueden alcanzar el 97% fcilmente. Echa un vistazo a nuestro punto de referencia de lado a lado para Fashion-MNIST vs. MNIST, y lee "La mayora de los pares de dgitos MNIST se pueden distinguir bastante bien con solo un pxel".
  2. MNIST est sobreutilizado. En un hilo de Twitter de abril de 2017, el investigador cientfico de Google Brain y experto en aprendizaje profundo, Ian Goodfellow, hizo un llamamiento a los investigadores a alejarse del conjunto MNIST.
  3. MNIST no puede representar tareas CV modernas , como se seala en este hilo de Twitter de abril de 2017, el experto en aprendizaje profundo/autor de Keras, Franois Chollet.

Los investigadores introdujeron a Fashion-MNIST como un reemplazo del conjunto de datos MNIST. El nuevo conjunto de datos contiene imgenes de diversos artculos de ropa, como camisas, zapatos, abrigos y otros artculos de moda.

El dataset Fashion-MNIST comparte la misma estructura dividida de train-test como MNIST. Mientras que en el caso del conjunto de datos MNIST las etiquetas de clase eran dgitos del 0-9, Las etiquetas de clase para Fashion-MNIST son las siguientes:

Label Description

1 Trouser
2 Pullover
3 Dress
4 Coat

6 Shirt
7 Sneaker

9 Ankle boot

$k$ vecinos más cercanos¶

El primer algoritmo que utilizaremos para clasificar las imgenes de ropa es el $k$-nn. En este paso del anlisis del dataset ajustaremos dos hiperparmetros del algoritmo:

  1. $k$: el nmero de vecinos que se consideran para clasificar un nuevo ejemplo. Probaremos con todos los valores entre 1 y 10.
  2. pesos: importancia que se da a cada uno de los vecinos considerados. En este caso probaremos dos opciones:
    1. pesos uniformes, donde todos los vecinos se consideran igual
    2. pesos segn distancia, donde los vecinos ms cercanos tienen ms peso en la clasificacin que los vecinos ms lejanos.

Para decidir cules son los hiperparmetros ptimos utilizaremos una bsqueda de rejilla (grid search), es decir, entrenaremos un modelo para cada combinacin de hiperparmetros posible y la evaluaremos utilizando validacin cruzada (cross validation) con 4 particiones estratificadas. Posteriormente escogeremos la combinacin de hiperparmetros que mejor resultados haya dado.

def plot_confusion_matrix(cm, classes):
    cm = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis]
   
    cmap=plt.cm.Blues

    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)

    thresh = cm.max() / 2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        plt.text(j, i, format(cm[i, j], ".2f"),
                 horizontalalignment="center",
                 color="white" if cm[i, j] > thresh else "black")

    plt.tight_layout()
    plt.ylabel('True label')
    plt.xlabel('Predicted label')

Support Vector Machines¶

Como continuacin de la prctica, en esta segunda parte clasificaremos las imgenes de ropa utilizando el algoritmo SVM con el kernel radial. En este caso, en lugar de utilizar una bsqueda de rejilla para ajustar los hiperparmetros del algoritmo utilizaremos una bsqueda aleatoria, es decir, probaremos combinaciones de parmetros al azar. Los hiperparmetros a optimizar son:

  • C: el valor de penalizacin de los errores en la clasificacin. Marca el compromiso entre obtener el hiperplano con el mayor margen posible y clasificar el mximo nmero de ejemplos correctamente. Probaremos valores aleatorios distribuidos uniformemente entre 1 y 500.

  • gamma: coeficiente que multiplica la distancia entre dos puntos en el kernel radial. Probaremos valores aleatorios distribuidos uniformemente entre 0.001 y 0.1
    • </ul> Igual que en el caso anterior, para validar el rendimiento del algoritmo con cada combinacin de hiperparmetros utilizaremos validacin cruzada (cross-validation) con 4 particiones estratificadas.

def coste(preds, labels):
    p = np.argmax(preds, 1)
    l = np.argmax(labels, 1)
    errors = np.sum(p != l)
    trousers = np.where(l == 1)[0]
    errors_trousers = np.sum(p[trousers] != 1)
    
    return errors + errors_trousers

def plot_learning_curve(estimator, title, X, y, ylim=None, cv=None,
                        n_jobs=1, train_sizes=np.linspace(.1, 1.0, 5)):
    """
    Generate a simple plot of the test and training learning curve.

    Parameters
    ----------
    estimator : object type that implements the "fit" and "predict" methods
        An object of that type which is cloned for each validation.

    title : string
        Title for the chart.

    X : array-like, shape (n_samples, n_features)
        Training vector, where n_samples is the number of samples and
        n_features is the number of features.

    y : array-like, shape (n_samples) or (n_samples, n_features), optional
        Target relative to X for classification or regression;
        None for unsupervised learning.

    ylim : tuple, shape (ymin, ymax), optional
        Defines minimum and maximum yvalues plotted.

    cv : int, cross-validation generator or an iterable, optional
        Determines the cross-validation splitting strategy.
        Possible inputs for cv are:
          - None, to use the default 3-fold cross-validation,
          - integer, to specify the number of folds.
          - An object to be used as a cross-validation generator.
          - An iterable yielding train/test splits.

        For integer/None inputs, if ``y`` is binary or multiclass,
        :class:`StratifiedKFold` used. If the estimator is not a classifier
        or if ``y`` is neither binary nor multiclass, :class:`KFold` is used.

        Refer :ref:`User Guide <cross_validation>` for the various
        cross-validators that can be used here.

    n_jobs : integer, optional
        Number of jobs to run in parallel (default 1).
    """
    plt.figure()
    plt.title(title)
    # Las siguientes 2 lineas del cdigo original no se usarn:
    # if ylim is not None:
    #    plt.ylim(*ylim)
    plt.xlabel("Training examples")
    plt.ylabel("Score")
    train_sizes, train_scores, test_scores = learning_curve(
        estimator, X, y, cv=cv, n_jobs=n_jobs, train_sizes=train_sizes)
    train_scores_mean = np.mean(train_scores, axis=1)
    train_scores_std = np.std(train_scores, axis=1)

    plt.plot(train_sizes, train_scores_mean, 'o-', color="r",
